Ritchey&AI RES Encoder Decoder

This page encodes and decodes binary data using Ritchey Encoding Scheme v1. Full 4-character blocks start with a lowercase letter and contain one uppercase letter, one number, and one special character from !@#$%&?+. A final 3-character block is used only when needed.

Encode

When a checksum is selected, the input is hashed first. The lowercase hexadecimal checksum string is then encoded as text bytes, not as the raw digest bytes.

This applies only when SHA-256 or SHA-512 is selected. A value of 1 means hash the input once. Higher values rehash the previous lowercase hexadecimal checksum text.

The trimmed version is copied from the start of the full encoded output. It is useful for passwords, but it is not intended to be decodable.
